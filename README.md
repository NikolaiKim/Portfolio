  ### :mag_right: Тестирование веб-приложения для учителей от Skyeng

#### Описание: 
  <p>В расписание учителей добавлена новая функциональность — личные события. Преподаватель может использовать личные события для собственных встреч, которые не привязаны к ученикам платформы Skyeng. Для тестирования была составлена необходимая тестовая документация в формате тест-кейсов и чек-листов. По завершении тестирования были оформлены баг-репорты, а на основе результатов создан отчет с использованием метрик и выводом о готовности продукта к релизу.<p>

  <p>Цель проекта:
    протестировать новую функциональность и вынести решение о готовности продукта к релизу.<p>

<p>
  
#### Как решал:
<ol>
  <li>Сделал тест-план </li>
  <li>Подготовил тестовую документацию</li>
  <li>Провёл тестирование</li>
  <li>Написал отчёт о результатах тестирования</li>
</ol>
  
<p>
  
  #### Какие инструменты были применены:
  
  <ol>
<li>Miro для создания декомпозиции новой функциональности;</li>
<li>Qase, Sitechko (Checkvist) для написания тестовой документации;</li>
<li>Confluence для ведения тест-плана и отчета;</li>
<li>Jira для фиксирования баг-репортов;</li>
<li>Тестирование API.</li>
  </ol>

### :mag_right: Тестирование проекта бэкенд-части функционала "Доска объявлений"

#### Описание:
<p>Суть проекта заключается в реализации функционала бэкенда сайта. <img src="https://media.giphy.com/media/3iyKHMIKg5VWG6qHUm/giphy.gif" width="55"> </p>
  
<p>
Бэкенд-часть проекта предполагает выполнение следующего функционала:
  <ol>
    <li>Авторизация и аутентификация пользователей.</li>
    <li>Распределение ролей между пользователями (пользователь и админ).</li>
    <li>Восстановление пароля через электронную почту (не обязательно).</li>
    <li>CRUD для объявлений на сайте (админ может удалять или редактировать все объявления, а пользователи только свои).</li>
    <li>Под каждым объявлением пользователи могут оставлять отзывы.</li>
    <li>В заголовке сайта можно осуществлять поиск объявлений по названию.</li>
</ol>

> <a href="https://github.com/NikolaiKim/advertising_website">Ссылка на Github проектa с разработчиками</a>

</p>

<p>
  
  #### Какие инструменты были применены в тестировании:
  
  <ol>
<li>Miro - для создания декомпозиции новой функциональности;</li>
<li>Qase, Sitechco (Checkvist) - для написания тестовой документации;</li>
<li>Confluence - для ведения тест-плана и отчета;</li>
<li>Jira - для фиксирования баг-репортов;</li>
<li>Тестирование API через Postman;</li>
<li>База данных PostgreSQL;</li>
<li>PyCharm - для запуска локального сервера.</li>
  </ol>
